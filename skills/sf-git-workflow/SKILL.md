# SF Git Workflow — Agent

You run in a **forked context** — isolated from the user's conversation.
The user invoked you with: **$ARGUMENTS**.

Parse `$ARGUMENTS`:
- Extract `TICKET` = il codice ticket (es. CPQ-2150, TGL-846, ecc.)
- Determina `CMD_ONLY` analizzando il tono della richiesta:
  - **`CMD_ONLY = false`** (esegui in autonomia) se la frase è **imperativa o diretta**: aggiorna, fai, porta, crea, allinea, sincronizza, manda, pulisci, converti, ecc.
  - **`CMD_ONLY = true`** (restituisci solo il comando) se la frase è **in prima persona**: voglio, vorrei, devo, mi serve, ho bisogno, preferirei, posso avere, ecc.
  - **`CMD_ONLY = ASK`** se il tono è ambiguo e non riesci a determinarlo con certezza.

## Step 0: Config Setup

The current branch list and saved config were injected above.

**If config was found** (no `CONFIG_NOT_FOUND`):
→ Read it silently. Extract `BRANCH_MAIN`, `BRANCH_QUALITY`, `BRANCH_PRODUCTION`. Proceed to Step 1.

**If config was NOT found**:
→ Analyze the remote branches to infer the structure:
- Look for: `release`, `main`, `master`, `develop` → candidate for `BRANCH_MAIN`
- Look for: `quality`, `staging`, `uat`, `test` → candidate for `BRANCH_QUALITY`
- Look for: `production`, `prod`, `master` (if main exists) → candidate for `BRANCH_PRODUCTION`

If you can infer all three with confidence → proceed and save.
If ambiguous → ask the user ONCE with the candidates:

```
Ho trovato questi branch. Conferma la mappatura:
- Branch principale (feature partono da qui): [candidate]
- Branch quality/staging: [candidate]
- Branch production: [candidate]
```

Once confirmed, save to `.claude/git-workflow-config.md`:

```markdown

BRANCH_QUALITY=[value]

FEATURE_PREFIX=feature

```

## Step 1: Intent Recognition

Parse the intent from `$ARGUMENTS` (ignoring `TICKET` and `cmd`):

| Intent | Keywords | Operazione |
|---|---|---|
| `new-feature` | "feature", "nuovo branch", "crea branch", "inizia ticket" | Crea feature/TICKET da BRANCH_MAIN |
| `new-hotfix` | "hotfix", "fix urgente", "patch prod" | Crea hotfix/TICKET da BRANCH_PRODUCTION |
| `feature-sync` | "allinea feature", "aggiorna feature", "sync feature", "feature sync" | Merge BRANCH_MAIN → feature/TICKET |
| `to-quality` | "quality", "porta in quality", "manda in quality", "QA" | Feature sync + crea qualitymerge/TICKET allineato a BRANCH_QUALITY |
| `quality-sync` | "sync quality", "aggiorna qualitymerge", "quality sync" | Merge BRANCH_QUALITY → qualitymerge/TICKET |
| `to-prod` | "prod", "produzione", "prepara PR", "release" | Feature sync + push (prepara per PR verso BRANCH_MAIN) |
| `feature-to-hotfix` | "converti in hotfix", "promuovi a hotfix" | Crea hotfix/TICKET da feature/TICKET allineato a BRANCH_PRODUCTION |
| `clean-qualitymerge` | "pulisci qualitymerge", "elimina qualitymerge", "clean" | Cancella tutti i branch locali qualitymerge/* |

If intent is ambiguous, ask the user before proceeding.

## Step 2: Build the Command

Build the exact command string based on intent and config values.
Replace BRANCH_MAIN, BRANCH_QUALITY, BRANCH_PRODUCTION with values from config.

### new-feature
```
git checkout BRANCH_MAIN && git pull origin BRANCH_MAIN && git checkout -b feature/TICKET
```

### new-hotfix
```
git checkout BRANCH_PRODUCTION && git pull origin BRANCH_PRODUCTION && git checkout -b hotfix/TICKET
```

### feature-sync
```
git checkout feature/TICKET && git fetch origin && git merge origin/BRANCH_MAIN --no-edit && git push origin feature/TICKET
```

### to-quality
```
git checkout feature/TICKET && git fetch origin && git merge origin/BRANCH_MAIN --no-edit && git push origin feature/TICKET && git checkout -b qualitymerge/TICKET && git fetch origin && git merge origin/BRANCH_QUALITY --no-edit && git push -u origin qualitymerge/TICKET
```

### quality-sync
```
git checkout qualitymerge/TICKET && git fetch origin && git merge origin/BRANCH_QUALITY --no-edit && git push origin qualitymerge/TICKET
```

### to-prod
```
git checkout feature/TICKET && git fetch origin && git merge origin/BRANCH_MAIN --no-edit && git push origin feature/TICKET
```

### feature-to-hotfix
```
git checkout feature/TICKET && git fetch origin && git checkout -b hotfix/TICKET && git merge origin/BRANCH_PRODUCTION --no-edit && git push -u origin hotfix/TICKET
```

### clean-qualitymerge
```
git branch | grep 'qualitymerge/' | xargs git branch -D
```
(If no qualitymerge branches exist, say only: "Nessun branch qualitymerge da eliminare.")

## Step 3: Execute or Return

**If `CMD_ONLY` is true:**
Return ONLY the command string. No explanation, no markdown, no extra text.

**If `CMD_ONLY` is false:**
Execute the command.
- On success: brief confirmation ("Branch feature/TICKET creato da BRANCH_MAIN.")
- On conflict: show which files conflict, stop, do not force-push.
- On error: show the git error and suggest the fix.

**If `CMD_ONLY` is ASK (tono ambiguo):**
Chiedi all'utente con questa domanda esatta (una sola volta, niente altro):
> Vuoi che lo esegua io, o preferisci il comando da lanciare tu?

Attendi la risposta e poi procedi di conseguenza.
